What is React?

A JavaScript library for building user interfaces

One of the most popular JavaScript library with over

160,000 stars on GitHub

Not a framework

(unlike Angular, which is more opinionated)

Open-source project created by

Facebook

React is the view layer of an application based on

Model View Controller

React efficiently rerenders Single Page Application using Virtual DOM concept

Setup and Installation

You can use Create React App - an environment created by Facebook pre-configured with everything you need to build a React app

Prerequisites

You need Node 10.16.0 or later version on your local development machine (but it’s not required on the server).

Choose one of the following methods:

  • npx
  • npm
  • Yarn

(run the code in your terminal one directory up from where you want the project to live)

npx

package runner tool coming with npm 5.2+ and higher 


								npx create-react-app my-app

npm

npm init is available in npm 6+


								npm init react-app my-app

Yarn

yarn create is available in Yarn 0.25+ 


								yarn create react-app my-app
It will create a directory "my-app" inside the current folder with initial project structure and transitive dependencies
Inside the newly created project run some built-in commands: 

								npm start
or 

								yarn start
to run the app in development mode. Open http://localhost:3000 to view it in the browser
Once you run this command, a new window will popup at localhost:3000 with your new React app and further instructions

JSX

- is a syntax extension to JavaScript, that looks like combination of JS and HTML

React doesn’t require using JSX, but it is helpful as a visual aid when working with UI inside the JS code

Instead of artificially separating technologies by putting markup and logic in separate files

React separates concerns with loosely coupled units called “components” that contain both

React function component

- JS function - accepts a single “props” object argument with data - returns a React element

Props

When React sees an element representing a user-defined component, it passes JSX attributes and children to this component as a single object. We call this object “props”
