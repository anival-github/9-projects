What is React?

A JavaScript library for building user interfaces

One of the most popular JavaScript library with over

160,000 stars on GitHub

Not a framework

(unlike Angular, which is more opinionated)

- It is positioned as a library, but not a framework.

Open-source project created by

Facebook

- It was created by Facebook, and it is one of the advantages of this technologies, as you know, that it will be supported and updated in future.

React is the view layer of an application based on

Model View Controller

MVC generally imply that data should be delivered to users in the form suitable for users, but it should be stored and processed in the way suitable for business. So while developing an application we should separately organize business layer of an application which will process and store data and user interface which will display data for users. And we can effectively create such an application using React.

React efficiently rerenders Single Page Application using Virtual DOM concept

- React uses Virtual DOM concept which allows to effectively render single page application that is application having one html document and a script with logic of how to change it.

JSX

- is a syntax extension to JavaScript, that looks like combination of JS and HTML

React doesn’t require using JSX, but it is helpful as a visual aid when working with UI inside the JS code

So what is JSX. It is not required to use it, but it is really helpful while using react.

React function component

- JS function - accepts a single “props” object argument with data - returns a React element

Props

When React sees an element representing a user-defined component, it passes JSX attributes and children to this component as a single object. We call this object “props”
Is one single argument containing all the data passed from one React component to another. We can pass data to another react component by adding a new attribute when we call component in our conde, so the name of the attribute will be the name of property inside props, and data included into braces will be the value of the property inside props. We can call data inside the React component by calling props, dot and name of the attribute we passed when calling React component.
